# Space Game

## Описание задачи

В рамках задания требовалось реализовать игру на JavaFX.  
Вместо классической змейки была реализована сетевая игра про космические корабли.

Игроки управляют кораблями, летают по карте, стреляют друг в друга и подбирают бафы.

## Что реализовано

В проекте реализованы:
- клиент-серверная архитектура;
- несколько игроков;
- движение корабля с инерцией;
- движение вперед по направлению взгляда корабля;
- стрельба;
- здоровье игроков;
- бафы на карте;
- fullscreen режим;
- общий конфиг;
- запуск по localhost или по сети.

## Архитектура

Проект разделен на клиентскую и серверную части.

Сервер отвечает за:
- подключение игроков;
- хранение состояния мира;
- обработку движения;
- обработку выстрелов;
- обработку столкновений;
- рассылку состояния клиентам.

Клиент отвечает за:
- ввод игрока;
- подключение к серверу;
- отправку действий игрока;
- получение состояния игры;
- отрисовку карты, кораблей, пуль и интерфейса.

## Структура проекта

Основной код находится в:

src/main/java/ru/nsu/tumilevich

Основные пакеты:
- client — клиентская часть;
- server — серверная часть;
- model — игровые объекты;
- config — настройки игры.

## Конфигурация

Все основные настройки находятся в файле:

src/main/java/ru/nsu/tumilevich/config/Config.java

Там можно изменить:
- IP сервера;
- порт;
- режим localhost / network;
- размер мира;
- скорость корабля;
- инерцию;
- здоровье;
- скорость пуль;
- fullscreen.

## Настройка сети

Для локального запуска:

LOCALHOST_MODE = true;

Для запуска по сети:

LOCALHOST_MODE = false;
NETWORK_SERVER_IP = "IP_СЕРВЕРА";

Например:

NETWORK_SERVER_IP = "100.100.154.104";


## Проверка сети

На Windows можно проверить подключение к серверу:

Test-NetConnection IP_СЕРВЕРА -Port 5555

Если выводит:

TcpTestSucceeded : True

значит клиент может подключиться к серверу.

## Возможные ошибки

Connection refused — сервер не запущен или указан неправильный IP/порт.

TcpTestSucceeded : False — проблема с сетью, firewall или VPN.

Если игра запускается локально, но не по сети, нужно проверить Config.java и порт 5555.
